# BookableCRM — WordPress Contact Form Widget

## What you're building

A full end-to-end integration that lets BookableCRM users embed a contact form on any WordPress site. Enquiries submitted through the form land directly in their BookableCRM pipeline.

The integration has four parts:
1. A Supabase DB migration (add `phone` to enquiries, document embed key/config pattern)
2. A Supabase Edge Function — the public HTTP endpoint the form POSTs to
3. A WordPress plugin — installable `.zip` with a shortcode
4. A new CRM settings page — where users configure the widget and copy their embed key

---

## Codebase context

**Stack:** React + TypeScript, Supabase (Postgres + Edge Functions), Vercel.

**Key existing patterns to follow:**
- Edge Functions live in `supabase/functions/<name>/index.ts`, written in Deno. See `supabase/functions/send-email/index.ts` for the full pattern — it uses `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` from `Deno.env.get()`, creates a service-role admin client, has CORS headers, and returns JSON responses.
- Migrations live in `supabase/migrations/` numbered sequentially (current highest is `013_activities_metadata.sql`). Next is `014_`.
- The `enquiries` table schema (from the type at `src/types/index.ts`):
  ```ts
  interface Enquiry {
    id: string
    tenant_id: string | null
    name: string
    email: string
    message: string
    source: string
    status: EnquiryStatus  // 'new' | 'contacted' | 'converted' | 'dismissed'
    notes: string | null
    pipeline_stage_id: string | null
    created_at: string
  }
  ```
  There is no `phone` field yet.
- The `tenants` table has a `settings` JSONB column (type `Record<string, unknown>`). Tenant-specific config (e.g. `email_config`) lives here. We'll use this for `embed_key` and `widget_config` too.
- There is an `enquiry_auto_stage` trigger that auto-assigns new enquiries to the tenant's first pipeline stage when `tenant_id` is set.
- RLS is active. The enquiries table has no public INSERT policy. The edge function must use the service-role client to bypass RLS.
- React components for settings live in `src/pages/` and `src/components/`. Look at any existing settings page for UI patterns (Tailwind, existing component library).

## Part 2 — Edge Function (`supabase/functions/submit-enquiry/index.ts`)

This is a **public** endpoint — no Authorization header required from callers. Authentication is via the `embed_key` in the request body.

**Request body (POST):**
```json
{
  "embed_key": "uuid-from-tenant-settings",
  "name": "Jane Smith",
  "email": "jane@example.com",
  "phone": "07700900000",
  "message": "I'd like to know more."
}
```

**Logic:**
1. Handle `OPTIONS` preflight (CORS — allow `*`).
2. Parse body. If `embed_key` is missing, return 400.
3. Using the **service-role** admin client, query `tenants` where `settings->>'embed_key' = embed_key`. If not found, return 401.
4. Read `widget_config` from the tenant's settings. Validate that required fields (email is always required; others depend on config) are present.
5. INSERT into `enquiries` with `tenant_id` from the matched tenant, `source = 'website'`, `status = 'new'`. The auto-stage trigger will handle `pipeline_stage_id`.
6. Return `{ success: true }` on 200.
7. On any error, return appropriate status + `{ error: "..." }`.

Follow the same Deno + esm.sh import style as `send-email/index.ts`.

## Part 3 — WordPress Plugin

Create a self-contained WordPress plugin at `wordpress-plugin/bookable-crm-form/bookable-crm-form.php`.

Also create `wordpress-plugin/bookable-crm-form.zip` — a zip of that directory, ready for WordPress → Plugins → Add New → Upload.

**Plugin spec:**

- **Plugin header:**
  ```
  Plugin Name: BookableCRM Contact Form
  Description: Embed a BookableCRM enquiry form on any page using the [bookable_crm_form] shortcode.
  Version: 1.0.0
  Author: Bookable Ltd
  ```

- **Settings page** (under Settings → BookableCRM Form):
  - Field: "Supabase Project URL" (e.g. `https://xyz.supabase.co`) — stored in `wp_options` as `bcrm_supabase_url`
  - Field: "Embed Key" (the UUID from their BookableCRM settings) — stored as `bcrm_embed_key`
  - These are the only two things the WordPress admin needs to configure.

- **Shortcode `[bookable_crm_form]`:**
  - Enqueues inline CSS and a small inline JS block (no external dependencies).
  - On page load, fetches `{supabase_url}/functions/v1/get-widget-config?embed_key={embed_key}` to get the `widget_config` (title, fields, labels) — see Part 4 below.
  - Renders the form fields defined in the config. Always renders email. Renders name/phone/message only if enabled in config.
  - On submit: POSTs JSON to `{supabase_url}/functions/v1/submit-enquiry`.
  - Shows a success message (from config) or a generic error.
  - The form should be styled neutrally — clean, minimal, easy to override with WordPress theme CSS. Use a `.bcrm-form` wrapper class.
  - No jQuery dependency. Vanilla JS only.

- **Security:** Nonce on the settings form, `sanitize_text_field` on inputs, `esc_attr` on output.

## Part 4 — get-widget-config Edge Function (`supabase/functions/get-widget-config/index.ts`)

A lightweight **public GET** endpoint the WordPress plugin calls to fetch the form configuration.

**Request:** `GET /functions/v1/get-widget-config?embed_key=<uuid>`

**Response:**
```json
{
  "title": "Get in touch",
  "fields": { "name": true, "email": true, "phone": false, "message": true },
  "submit_label": "Send enquiry",
  "success_message": "Thanks! We'll be in touch shortly."
}
```

**Logic:** Look up tenant by `embed_key` (service-role client), return the `widget_config` from settings, with sensible defaults if the key hasn't been set up yet. Return 401 if embed_key not found.

## Part 5 — CRM Settings Page (React)

Create a new settings tab/page within BookableCRM where users manage their widget.

**Location:** Add it to the existing settings area. Look at how other settings pages are structured and follow the same pattern.

**UI sections:**

1. **Embed Key**
   - Display the current embed key (or "Not generated yet").
   - "Generate new key" button — calls a Supabase RPC or direct update to set `settings->embed_key` to a new `gen_random_uuid()`. Warn that regenerating will break existing embeds.
   - Copy-to-clipboard button.

2. **Form Fields**
   - Toggle for each field: Name, Phone, Message (email is always on and shown as locked).
   - Text inputs for: Form title, Submit button label, Success message.
   - "Save" button — updates `tenants.settings.widget_config` via Supabase client.

3. **Embed Instructions**
   - **WordPress:** Step 1: install the plugin (link to download or note the zip). Step 2: enter Supabase URL + Embed Key in Settings → BookableCRM Form. Step 3: add `[bookable_crm_form]` to any page.
   - **Other sites (embed snippet):** Show a copyable `<script>` snippet:
     ```html
     <div id="bcrm-form"></div>
     <script src="https://<project>.supabase.co/functions/v1/widget-embed.js?key=<embed_key>"></script>
     ```
     (Note: this JS file doesn't need to exist yet — just show the snippet so users know the roadmap. Mark it as "coming soon" if you want to be accurate.)
